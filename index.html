Lesson 1.1: The Basics of the Web and HTML

How does the web work?

Say I type in www.espn.com to check the score of the Wisconsin Badger game. My computer sends a request to a server. The server finds the HTML document I've requested, and sends it back to my web browser, which then interprets the document and displays the information on my screen.

What is HTML?

HTML stands for Hyper Text Markup Language. These documents are the majority of what you see on the web. HTML documents contain text content which describes "what you see" and markup which describes "how it looks".

How do computers interpret information we give them?

As our instructor stated, "computers are stupid." What he meant is that computers take the instructions we give them literally. This can make a tiny mistake into a huge problem, but it also gives us powerful control over what we can instruct them to do.

What are tags and elements?

HTML documents are made of elements. When writing HTML, we tell browsers the type of each element by using tags. To make matters even more confusing, there are two different types of elements: inline and block. A much more detailed description can be found by clicking here.

Lesson 1.2: Creating a Structured Document

What's DOM?

Dominant Object Model, or DOM, is the tree like structure of HTML. This comes from the fact that HTML elements can have other elements inside of them.

How is the web-page structured?

When viewing the intra-structure of a website, you'll find that everything on the webpage is rectangular. If you go deep enough into the structure, you'll also find the same text that is displayed on the webpage. To see this, visit wikipedia, turn on the developer tools and check it out for yourself!

What tools can we use?

There are many tools programmer's have at their disposal. A few that we will use over the course of the program are Scratchpen, Codepen, and Sublime Text.

Lesson 1.3: Adding CSS Style to HTML Structure

What is CSS?

CSS stands for Cascading Style Sheets. The purpose of CSS is to give programmers a way to control the style of related HTML elements. This is done by giving similar HTML elements the same class name and then specifying the style that should apply to that class.

How does this work?

Here's an example of how to use CSS.

If you want to write CSS that makes all h1 elements have a black background and white text, you'd write:

h1 {background-color: black;color: white;}

What is The Box Model?

As I previously mentioned, everything on a webpage is in a box form. The box contains more than just the content though. The content is just the inner layer of the box, surrounded by the padding; border; and finally the margin. For a visual example, click here.

Why avoid repetition?

The ability to avoid repetition in coding is extremely important for several reasons.

Time & Errors: Failing to avoid repetition will not only waste hours of your time, but it can lead to errors. If you need to make a chage to something, you would need to go back and make the same change to all the same aspects of the code. Forgetting to do so can lead to headaches!

Consistency: Instead of manually giving each individual one the same style, it is possible to give each div the same class and then specify the style for that class in only one place! By avoiding repetion, it's now very easy to make changes to this style.
